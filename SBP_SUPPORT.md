# 💳 Отчёт о добавлении поддержки СБП

**Дата:** 19 апреля 2026
**Статус:** Backend и модели готовы ✅ | UI в разработке

## Выполненные изменения

### 1. ✅ База данных
**Создан enum тип:**
```sql
CREATE TYPE payment_method_enum AS ENUM ('sbp', 'card');
```

**Добавлены поля в таблицу fundraisers:**
- `payment_method` - метод оплаты (sbp/card), NOT NULL
- `sbp_phone` - номер телефона для СБП
- `sbp_bank` - банк для СБП
- `card_number` - теперь nullable
- `card_holder_name` - теперь nullable

**Добавлен constraint:**
```sql
CHECK (
  (payment_method = 'sbp' AND sbp_phone IS NOT NULL AND sbp_bank IS NOT NULL) OR
  (payment_method = 'card' AND card_number IS NOT NULL)
)
```

**Миграция:** `002_add_sbp_support.sql`

### 2. ✅ Backend API
**Обновлён fundraiserController.js:**
- Добавлена обработка `payment_method`
- Валидация полей в зависимости от метода:
  - СБП: требует `sbp_phone` и `sbp_bank`
  - Карта: требует `card_number`
- Сохранение всех полей в БД

**API endpoint:** `POST /api/fundraisers`
```json
{
  "title": "Название сбора",
  "description": "Описание",
  "category": "medical",
  "goal_amount": 100000,

}
```

### 3. ✅ Flutter модели
**Обновлена модель Fundraiser:**
