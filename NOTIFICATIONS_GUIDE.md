# Система уведомлений

## Обзор

Система уведомлений автоматически информирует пользователей о важных событиях в приложении.

## Типы уведомлений

### 1. **donation_received** 💰
Отправляется владельцу сбора, когда кто-то делает пожертвование.
- **Когда:** При создании нового доната
- **Кому:** Владелец сбора
- **Пример:** "Вам пришло пожертвование на сумму 2000₽. Проверьте и подтвердите."

### 2. **donation_approved** ✅
Отправляется донору, когда его пожертвование подтверждено.
- **Когда:** При подтверждении доната владельцем сбора
- **Кому:** Донор
- **Пример:** "Ваше пожертвование на сумму 2000₽ подтверждено. Теперь вы можете создать свой сбор!"

### 3. **donation_rejected** ❌
Отправляется донору, когда его пожертвование отклонено.
- **Когда:** При отклонении доната владельцем сбора
- **Кому:** Донор
- **Пример:** "К сожалению, ваше пожертвование было отклонено. Попробуйте снова."

### 4. **achievement_earned** 🏆
Отправляется при получении нового достижения.
- **Когда:** При выполнении условий достижения
- **Кому:** Пользователь, получивший достижение
- **Пример:** "Достижение: Первый шаг - Совершите первое пожертвование"

## Реализация

### Backend

**Создание уведомления:**
```javascript
await pool.query(
  `INSERT INTO notifications (user_id, type, title, message, related_id)
   VALUES ($1, $2, $3, $4, $5)`,
  [userId, 'donation_received', 'Новое пожертвование!', 'Сообщение', donationId]
);
```

**API endpoints:**
- `GET /api/notifications` - Получить все уведомления
- `PATCH /api/notifications/:id/read` - Отметить как прочитанное
- `PATCH /api/notifications/read-all` - Отметить все как прочитанные

### Frontend (Flutter)

**NotificationProvider:**
- `loadNotifications()` - Загрузить уведомления
- `markAsRead(id)` - Отметить уведомление как прочитанное
- `markAllAsRead()` - Отметить все как прочитанные
- `unreadCount` - Количество непрочитанных уведомлений

**Использование:**
```dart
// Загрузить уведомления
context.read<NotificationProvider>().loadNotifications();

// Получить количество непрочитанных
final unreadCount = context.watch<NotificationProvider>().unreadCount;

// Отметить как прочитанное
await provider.markAsRead(notificationId);
```

## UI компоненты

### Badge с количеством непрочитанных
В `HomeScreen` на иконке уведомлений отображается красный badge с количеством непрочитанных уведомлений.

### Экран уведомлений
- Список всех уведомлений
- Непрочитанные выделены синим фоном
- Показывается относительное время (например, "5 минут назад")
- Pull-to-refresh для обновления
- Кнопка "Прочитать все" в AppBar

## Автоматическое создание уведомлений

Уведомления создаются автоматически в следующих случаях:

1. **При создании доната** (`donationController.js:36-46`)
   - Уведомление владельцу сбора о новом пожертвовании

2. **При подтверждении доната** (`donationController.js:125-136`)
   - Уведомление донору о подтверждении
   - Проверка и уведомления о новых достижениях

3. **При отклонении доната** (`donationController.js:216-225`)
   - Уведомление донору об отклонении

4. **При получении достижения** (`donationController.js:159-163`)
   - Уведомление о новом достижении

## Тестирование

### Создать тестовое уведомление:
```bash
cd backend
node -e "const pool = require('./src/config/database'); \
  pool.query('INSERT INTO notifications (user_id, type, title, message) \
  VALUES (1, \"donation_received\", \"Тест\", \"Тестовое уведомление\")') \
  .then(() => console.log('OK')).finally(() => process.exit())"
```

### Проверить API:
```bash
curl -X GET http://127.0.0.1:3003/api/notifications \
  -H "Authorization: Bearer YOUR_TOKEN"
```
