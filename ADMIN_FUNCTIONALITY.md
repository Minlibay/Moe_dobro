# Функционал Админ-панели "Моё добро"

## Обзор

Платформа "Моё добро" имеет две админ-панели:
1. **Веб-админка** (React) - `admin-web/` - полнофункциональная панель для администраторов
2. **Мобильная админка** (Flutter) - встроена в мобильное приложение

---

## 🌐 Веб-админка (React)

### Технологии
- **Frontend**: React 18 + Vite
- **UI**: Tailwind CSS + Lucide Icons
- **Графики**: Recharts
- **HTTP**: Axios
- **Роутинг**: React Router v6

### Структура приложения

```
admin-web/
├── src/
│   ├── components/
│   │   ├── Dashboard.jsx      # Главная панель со статистикой
│   │   ├── Users.jsx          # Управление пользователями
│   │   ├── Fundraisers.jsx    # Управление сборами
│   │   ├── Donations.jsx      # Управление донатами
│   │   └── Login.jsx          # Авторизация
│   ├── services/
│   │   └── api.js             # API клиент
│   ├── App.jsx                # Главный компонент
│   └── main.jsx               # Точка входа
```

## 📊 1. Dashboard (Панель управления)

### Функционал

#### Статистика (карточки)
- **Пользователи**: Общее количество + верифицированные
- **Сборы**: Общее количество + активные
- **Донаты**: Общее количество + ожидающие подтверждения
- **Финансы**: Всего собрано + средний донат
- **Одобренные донаты**: Количество
- **Ожидающие донаты**: Количество

#### Графики
1. **График донатов за 30 дней** (Line Chart)
   - Количество донатов по дням
   - Сумма донатов по дням
   - Интерактивный с Tooltip

2. **Популярные категории** (Pie Chart)
   - Распределение сборов по категориям
   - Цветовая кодировка
   - Интерактивный

#### Таблица категорий
- Название категории (Ипотека, Лечение, Образование, Другое)
- Количество сборов
- Общая сумма собранных средств

## 👥 2. Users (Управление пользователями)

### Функционал

#### Поиск и фильтрация
- Поиск по имени или телефону
- Поиск в реальном времени (Enter для применения)

#### Таблица пользователей
Отображаемые поля:
- **ID**: Уникальный идентификатор
- **Имя**: Полное имя пользователя
- **Телефон**: Номер телефона
- **Донатов**: Общая сумма пожертвований (₽)
- **Получено**: Общая сумма полученных средств (₽)
- **Статус**:
  - 🔴 Заблокирован
  - 🟢 Верифицирован
  - ⚪ Не верифицирован

#### Действия с пользователями

1. **Верификация**
   - Кнопка "Верифицировать" / "Снять"
   - Верифицированные пользователи получают галочку
   - Заблокированных пользователей нельзя верифицировать

2. **Блокировка**
   - Кнопка "Заблокировать" / "Разблокировать"
   - При блокировке требуется указать причину
   - Заблокированные пользователи не могут:
     - Создавать сборы
     - Делать донаты
     - Быть верифицированными

### API эндпоинты

```javascript
// Получить список пользователей
GET /api/admin/users?search=Иван&limit=100

// Верифицировать пользователя
PATCH /api/admin/users/:userId/verify
Body: { "is_verified": true }

// Заблокировать пользователя
PATCH /api/admin/users/:userId/block
Body: {
  "is_blocked": true,
  "block_reason": "Нарушение правил"
}
```

## ❤️ 3. Fundraisers (Управление сборами)

### Функционал

#### Фильтры
- **Поиск**: По названию сбора
- **Статус**:
  - Все статусы
  - Активные
  - Завершенные
  - Закрытые
- **Категория**:
  - Все категории
  - Ипотека
  - Лечение
  - Образование
  - Другое

#### Таблица сборов
Отображаемые поля:
- **ID**: Уникальный идентификатор
- **Название**: Название сбора + ⭐ (если избранный)
- **Категория**: Ипотека/Лечение/Образование/Другое
- **Цель**: Целевая сумма (₽)
- **Собрано**: Текущая сумма (₽)
- **Статус**:
  - 🟢 Активный
  - 🔵 Завершен
  - ⚪ Закрыт
- **Завершение**:
  - ✅ Подтверждено (зеленый бейдж)
  - 👁️ Проверить (желтый бейдж, кликабельный)
  - Не отправлено (серый текст)

#### Действия со сборами (меню ⋮)

1. **Закрыть сбор**
   - Доступно только для активных сборов
   - Требует подтверждения
   - Сбор становится недоступен для донатов

2. **Избранное**
   - Добавить/убрать из избранного
   - Избранные сборы отображаются первыми в приложении
   - Помечены звездочкой ⭐

3. **Удалить**
   - Полное удаление сбора
   - Требует подтверждения
   - Необратимое действие

#### Модальное окно подтверждения завершения

Когда создатель сбора отправляет подтверждение завершения, админ видит:

**Информация:**
- Создатель (имя + телефон)
- Цель и собранная сумма
- Дата отправки

**Сообщение от создателя:**
- Текстовое описание как были использованы средства

**Подтверждающий документ:**
- Фото/скриншот (чек, договор, квитанция и т.д.)
- Полноразмерный просмотр

**Действия:**
- ✅ Подтвердить завершение - сбор получает статус "подтверждено"
- Закрыть - вернуться к списку

### API эндпоинты

```javascript
// Получить список сборов
GET /api/admin/fundraisers?search=ипотека&status=active&category=mortgage&limit=100

// Закрыть сбор
PATCH /api/admin/fundraisers/:fundraiserId/close

// Добавить/убрать из избранного
PATCH /api/admin/fundraisers/:fundraiserId/feature
Body: { "is_featured": true }

// Подтвердить завершение
PATCH /api/admin/fundraisers/:fundraiserId/verify-completion

// Удалить сбор
DELETE /api/admin/fundraisers/:fundraiserId
```

## 💰 4. Donations (Управление донатами)

### Функционал

#### Фильтры
- **Статус**:
  - Все статусы
  - Ожидают (pending)
  - Одобрены (approved)
  - Отклонены (rejected)

#### Таблица донатов
Отображаемые поля:
- **ID**: Уникальный идентификатор
- **От кого**: Имя донора + телефон
- **Кому**: Имя получателя
- **Сбор**: Название сбора (truncated)
- **Сумма**: Сумма доната (₽)
- **Скриншот**: Кнопка "👁️ Просмотр"
- **Статус**:
  - 🟢 Одобрен
  - 🟡 Ожидает
  - 🔴 Отклонен
- **Дата**: Дата и время создания
- **Действия**: Кнопки одобрения/отклонения (только для pending)

#### Визуальные индикаторы
- **Желтый фон строки**: Донат ожидает более 24 часов (требует внимания!)

#### Модальное окно просмотра скриншота

**Информация о донате:**
- От кого (имя + телефон)
- Кому (имя + телефон)
- Сумма
- Название сбора

**Реквизиты для перевода** (голубой блок):

Для СБП:
- Способ оплаты: СБП
- Номер телефона (крупным шрифтом, моноширинный)
- Банк
- 💡 Подсказка: "Донор должен был перевести деньги через СБП на этот номер"

Для карты:
- Способ оплаты: Банковская карта
- Номер карты (крупным шрифтом, моноширинный)
- Владелец карты
- Банк
- 💡 Подсказка: "Донор должен был перевести деньги на эту карту"

**Скриншот перевода:**
- Полноразмерное изображение
- Fallback при ошибке загрузки

**Действия:**
- ✅ Одобрить - донат засчитывается, обновляется статистика
- ❌ Отклонить - требуется указать причину
- Закрыть - вернуться к списку

#### Процесс одобрения доната

При одобрении происходит:
1. Статус доната меняется на "approved"
2. Сумма добавляется к `current_amount` сбора
3. Обновляется статистика донора:
   - `total_donated` += сумма
   - `people_helped` += 1
4. Обновляется статистика получателя:
   - `total_received` += сумма
5. Проверяются и начисляются достижения донору
6. Отправляется уведомление донору

### API эндпоинты

```javascript
// Получить список донатов
GET /api/admin/donations?status=pending&limit=100

// Одобрить донат
PATCH /api/admin/donations/:donationId/approve

// Отклонить донат
PATCH /api/admin/donations/:donationId/reject
Body: { "reason": "Неверная сумма на скриншоте" }
```

## 📱 Мобильная админка (Flutter)

### Доступ
- Встроена в мобильное приложение
- Доступна только пользователям с `is_admin = true`
- Путь: Профиль → Админ-панель (карточка с 👑)

### Структура

4 вкладки:
1. **📊 Статистика** - общая статистика платформы
2. **👥 Пользователи** - управление пользователями
3. **❤️ Сборы** - управление сборами
4. **💰 Донаты** - просмотр донатов

### Функционал

#### 1. Статистика
- Карточки с основными метриками
- Финансовая сводка
- Pull-to-refresh

#### 2. Пользователи
- Список всех пользователей
- Информация: имя, телефон, донаты, получено
- Действия:
  - ✅ Верифицировать / снять верификацию
  - Статус отображается иконкой

#### 3. Сборы
- Список всех сборов
- Информация: название, сумма, статус
- Действия (PopupMenu):
  - Закрыть сбор
  - Добавить/убрать из избранного
  - Удалить (с подтверждением)

#### 4. Донаты
- Список всех донатов
- Информация: сумма, статус, дата
- Цветовые индикаторы статуса

### Отличия от веб-версии
- ❌ Нет графиков (ограничения мобильного UI)
- ❌ Нет модальных окон просмотра скриншотов
- ❌ Нет одобрения/отклонения донатов (только просмотр)
- ❌ Нет блокировки пользователей
- ✅ Более компактный интерфейс
- ✅ Pull-to-refresh на всех экранах

## 🔒 Безопасность

### Аутентификация
- JWT токены с истечением срока действия
- Токен хранится в localStorage (веб) / SharedPreferences (мобильное)
- Автоматический logout при 401 ошибке

### Авторизация
- Проверка `is_admin = true` на бэкенде
- Middleware `authMiddleware` на всех admin роутах
- Проверка прав на фронтенде (ProtectedRoute)

### Валидация
- Подтверждение деструктивных действий (удаление, блокировка)
- Обязательные поля (причина блокировки, причина отклонения)

## 🔄 Workflow админа

### Ежедневные задачи
1. Проверить Dashboard - общая ситуация
2. Обработать ожидающие донаты (желтые строки!)
3. Проверить новые подтверждения завершения
4. Ответить на жалобы (если есть система жалоб)

### Еженедельные задачи
1. Проверить статистику по категориям
2. Выбрать избранные сборы
3. Проверить неактивных пользователей
4. Анализ трендов донатов

### По запросу
1. Верификация пользователей
2. Блокировка нарушителей
3. Закрытие сборов
4. Удаление неподходящего контента
