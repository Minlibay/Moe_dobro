# Вкладка "Завершенные сборы" - Документация

## Что добавлено

### 1. Backend изменения

#### База данных
- Добавлены поля в таблицу `fundraisers`:
  - `completion_proof_url` - URL отчетного документа (скриншот/фото)
  - `completion_message` - Сообщение организатора о завершении
  - `completion_submitted_at` - Дата отправки отчета
  - `completion_verified` - Подтвержден ли отчет администратором
  - `completion_verified_at` - Дата подтверждения
  - `completion_verified_by` - ID администратора, подтвердившего отчет

#### API
- Обновлен endpoint `GET /api/fundraisers?status=verified_completed`
  - Возвращает только завершенные сборы с подтвержденными отчетами
  - Сортировка по дате подтверждения (новые сверху)
  - Поддержка фильтрации по категориям

### 2. Frontend изменения (Flutter)

#### Новый виджет
- `CompletedFundraiserCard` - специальная карточка для завершенных сборов
  - Отображает отчетный документ (изображение)
  - Показывает категорию сбора с эмодзи
  - Выводит сообщение о завершении
  - Статистика: собранная сумма и количество донатов
  - Дата завершения в удобном формате

#### Обновлен HomeScreen
- Добавлена третья вкладка "Завершенные" в BottomNavigationBar
- Автоматическая загрузка завершенных сборов при переключении на вкладку
- Красивый пустой экран, если завершенных сборов нет

#### Provider
- Метод `loadCompletedFundraisers()` для загрузки завершенных сборов
- Хранение списка в `_completedFundraisers`

## Как это работает

### Для пользователей

1. **Просмотр завершенных сборов**
   - Открыть приложение
   - Перейти на вкладку "Завершенные" (третья иконка снизу)
   - Увидеть список всех подтвержденных завершенных сборов

2. **Что видно в карточке**
   - Отчетный документ (фото/скриншот подтверждения)
   - Категория сбора (🏠 Ипотека, 💊 Лечение, 📚 Образование, 🎯 Другое)
   - Название сбора
   - Сообщение организатора о результатах
   - Сколько денег собрано
   - Сколько человек помогло
   - Когда завершен

### Для организаторов сборов

1. **Завершение сбора**
   - Закрыть сбор через API: `PATCH /api/fundraisers/:id/close`
   - Статус меняется на `completed`

2. **Отправка отчета**
   - Загрузить отчетный документ: `POST /api/fundraisers/:id/completion-proof`
   - Прикрепить фото/скриншот подтверждения
   - Написать сообщение о результатах

3. **Подтверждение администратором**
   - Администратор проверяет отчет
   - Устанавливает `completion_verified = true`
   - Сбор появляется во вкладке "Завершенные"

## Приоритет отображения

Завершенные сборы **всегда в самом низу по приоритету**:
- Активные сборы показываются на главной вкладке
- Завершенные сборы изолированы в отдельной вкладке
- Внутри вкладки "Завершенные" сортировка по дате подтверждения (новые сверху)

## API Endpoints

### Получить завершенные сборы
```
GET /api/fundraisers?status=verified_completed
```

**Query параметры:**
- `category` (optional) - фильтр по категории: `mortgage`, `medical`, `education`, `other`
- `limit` (optional, default: 20) - количество записей
- `offset` (optional, default: 0) - смещение для пагинации

**Ответ:**
```json
[
  {
    "id": 1,
    "title": "Помощь на лечение мамы",
    "description": "...",
    "category": "medical",
    "goal_amount": "150000.00",
    "current_amount": "150000.00",
    "completion_proof_url": "/uploads/medical_completion.jpg",
    "completion_message": "Операция прошла успешно!",
    "completion_verified": true,
    "completion_verified_at": "2026-04-16T11:10:30.726Z",
    "creator_name": "Иван Иванов",
    "donors_count": 25,
    "progress_percent": 100.00
  }
]
```
